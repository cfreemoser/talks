Word2Vec

Word2Vec is an algorithm that accepts text corpus as an input and outputs a vector representation for each word. It consists of a two-layer neural net, which learns the semantic behind the word.

Word2Vec training process

Two ways how we can learn semantic

  • Skip-gram

    Given a set of sentences, the model loops on the words of each sentence trying to use the current word w in order to predict its neighbors
    $$\text{w} \rightarrow \text{w}_{p+2}, \text{w} \rightarrow \text{w}_{p+1} , \text{w} \rightarrow \text{w}_{p-1}, \text{w} \rightarrow \text{w}_{p-2}$$

  • CBOW

    Given a set of sentences, the model loops on the words of each sentence trying to use the surrounding words to predict the current word w
    $$\text{w}_{p-2} \rightarrow \text{w}, \text{w}_{p-1} \rightarrow \text{w} , \text{w}_{p+2} \rightarrow \text{w}, \text{w}_{p+1} \rightarrow \text{w}$$

Vector-oriented reasoning

Doing Math with words

  • Singular/plural relation

    Word2Vec is able to understand the relation between the plural and the singular of words $$\left (\overrightarrow{cat} - \overrightarrow{cats} \right ) \approx \left (\overrightarrow{car} - \overrightarrow{cars} \right )$$

  • Distance between words

    Word2Vec can calculate the distance between words to find similarity $$\left \| \overrightarrow{dog} - \overrightarrow{cat} \right \| = 0.7130494$$

  • Answering analogy questions

    World analogies can be resolved with Word2Vec $$\overrightarrow{x} = \overrightarrow{men} + \left (\overrightarrow{woman} - \overrightarrow{queen} \right )$$

  • Element-wise addition

    Element-wise addition can be used to reveal relationships $$\overrightarrow{German} + \overrightarrow{airline} = \overrightarrow{Lufthansa}$$

Doc2Vec

We can extend the Word2Vec model by adding another vector, the so called document-unique, to learn the vector representation of whole documents.

Building a Model

How to compute a judgment

The Idea behind the Model

Features to base a judgment on

  • 1: Transform transcript with Doc2Vec
  • 2: Associate the transcript with the ten most referenced answers
  • 3: Calculate the distance between the transcript and the answers

Each prompt is associated with a set of reference answers to fulfill the given task. Using Doc2Vec, the distance between the child's response and the reference answers can be computed. Based on those distances a judgment can be made.

Architcture of the Classifier

A Binary classifier based on the distance of the ten most referenced answers to a prompt

  • Input : Distances between reference grammar and transcript
  • Layers: 2-Hidden Layers
  • Output : Binary Judgment - reject or accept
#Initializing Neural Network

classifier.add(Dropout(0.2))

Why we chose polyaxon

  • Versioning and reproducability

    Reproducable results with a built-in version control for code and experiments.

  • Hyperparameter search

    Run group of experiments in parallel and in a distributed way.

  • Runs on any infrastructure

    Deploy Polyaxon in the cloud, on-premises or in hybrid environments.

  • User resources allocation

    Manage your team's resources and parallelism quotas.

  • Scalable

    Polyaxon can easily scale horizontally by adding more nodes.

  • Powerful interface

    Author jobs, experiments, and pipelines in Json, YAML, and Python.

Polyaxon Concepts

Running your Models

  • Project

    A Project in Polyaxon is very similar to a project in GitHub, it aims at organizing your efforts to solve a specific problem. A project consist of a name and a description, the code to execute, the data and a polyaxonfile.yml.

  • Experiment

    An Experiment is the execution of your model with data and the provided parameters on the cluster. An Experiment Job is the Kubernetes pod running on the cluster for a specific experiment.

  • Experiment Group

    A selection of experiments to compare. It might be the same version of your model and data, but with different hyperparameters based on a hyperparameter search algorithm.
